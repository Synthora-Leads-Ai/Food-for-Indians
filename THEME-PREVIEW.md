## 🎬 Transition Experience

### When Theme Changes
1. **Subtle overlay** appears (0.3s)
2. **Colors transition** smoothly (1s)
3. **Notification slides in** (desktop only)
4. **3D scene updates** lighting
5. **All elements harmonize** together

### Transition Timeline
```
0.0s  → User continues browsing
0.3s  → Subtle white overlay fades in
0.5s  → Colors begin transitioning
1.0s  → New theme fully applied
1.5s  → Overlay fades out
3.0s  → Notification disappears
```

## 🎨 Design Principles

### Color Harmony
- Each theme uses 3 main colors
- Gradients blend all 3 colors
- Background contrasts with text
- Accent colors add visual interest

### Accessibility
- All themes maintain WCAG contrast ratios
- Text remains readable in all themes
- Interactive elements clearly visible
- Focus states use theme colors

### Performance
- CSS transitions (GPU accelerated)
- No layout shifts during transitions
- Smooth 60fps animations
- Minimal JavaScript overhead

## 🎭 User Experience Flow

### First Visit (Auto Mode)
1. Site loads with current time's theme
2. User browses naturally
3. If they stay long enough, theme changes automatically
4. Notification shows new theme name
5. Experience feels fresh and dynamic

### Manual Override
1. User clicks theme switcher (palette icon)
2. Sees all 4 theme options with previews
3. Selects preferred theme
4. Theme locks until they re-enable Auto
5. Can switch back to Auto anytime
